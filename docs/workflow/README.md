# workflow/

Documents that govern how development work is carried out on this project.

| File | Purpose |
|---|---|
| [`process.md`](process.md) | End-to-end milestone workflow — starting, working on, and closing issues; scope changes and deferrals |
| [`checklist.md`](checklist.md) | Checklists for milestone start, session start, before closing an issue, and milestone close |
| [`issue-audit.md`](issue-audit.md) | Periodic open issue audit — classify, verify, and formally close issues that were shipped but never closed |
| [`issues.md`](issues.md) | Required content for bug, enhancement, and research issues; architecture decision rules; mid-milestone discovery process |
| [`release.md`](release.md) | Patch and milestone release procedure — from branch to tagged release and issue closure |
| [`cve.md`](cve.md) | CVE handling lifecycle — assess, mitigate, dismiss Dependabot alert, archive when patched |
| [`cve-template.md`](cve-template.md) | Template for per-project CVE documents written during the CVE workflow |

These complement the pre-push checklist in `CLAUDE.md`. `CLAUDE.md` is the AI assistant's reference; the files here are the human-facing working documents.
